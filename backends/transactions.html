stdnet.transaction(*models, **kwargs)¶

Create a transaction

Transaction class¶

class stdnet.Transaction(server, cursor, name=None)¶

Transaction class for pipelining commands to the back-end server. An instance of this class is usally obtained by using the high level stdnet.transaction() function.

name¶

Transaction name

server¶

Instance of a stdnet.BackendDataServer to which the transaction is beeing performed.

cursor¶

The server cursor which manages the transaction.

add(func, args, kwargs, callback=None)¶

Add an new operation to the transaction.

Parameters:
  • func – function to call which accept stdnet.Transaction.cursor as its first argument.
  • args – tuple or varying arguments to pass to func.
  • kwargs – dictionary or key-values arguments to pass to func.
  • callback – optional callback function with arity 1 which process the result wonce back from the server.
merge(other)¶

Merge two transaction together

empty()¶

Check if the transaction contains query data or not. If there is no query data the transaction does not perform any operation in the database.

structure_pipe(structure)¶

Create a pipeline for a structured datafield

commit()¶

Close the transaction and commit commands to the server.

get_result()¶

Retrieve the result after the transaction has executed. This can be done once only.
