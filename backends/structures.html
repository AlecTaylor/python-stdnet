Remote Data Structures¶

Data structures are fundamental constructs around which you build your application. They are used in almost all computer programs, therefore becoming fluent in what the standard data-structures can do for you is essential to get full value out of them. The standard template library in C++ implements a wide array of structures, python has several of them too. stdnet implements five remote structures: list, set, ordered set, hash and timeseries.

The structures are bind to a remote dataserver and they derive from from stdnet.Structure base class.

Creating Structures¶

The struct object¶

stdnet.struct¶

Creating the five structures available in stdnet is accomplished by using struct object in the following way:

from stdnet import struct

l = struct.list()
s = struct.set()
o = struct.zset()
h = struct.hash()
t = struct.ts()

Low level API¶

Structure Base Class¶

class stdnet.Structure(server, id, pipetype, instance=None, timeout=0, pickler=None, value_pickler=None, scorefun=None, **kwargs)¶

Base class for remote data-structures. Remote structures are the backend of structured fields but they can also be used as stand alone objects. For example:

import stdnet
db = stdnet.getdb(...)
mylist = db.list('bla')
Parameters:
  • server – instance of the remote server where the structure is stored.
  • id – structure id
  • pipetype – check the pipetype. Specified by the server.
  • instance – Optional stdnet.orm.StdModel instance to which the structure belongs to via a structured field. This field is specified when accessing remote structures via the object relational mapper.
server¶

instance of a stdnet.BackendDataServer.

pipetype¶

one of list, hash, ordered_set, unordered_set, ts.

id¶

unique id for the structure

instance¶

An optional stdnet.orm.StdModel instance to which the structure belongs to via a structured field.

Defaulr None.

timeout¶

Expiry timeout. If different from zero it represents the number of seconds after which the structure is deleted from the data server.

Default 0.

pickler¶

Class used for serialize and unserialize values. If None the server pickler will be used.

Default None.

scorefun¶

A callable which takes a value as parameter and return a float number to be usedto score the value. Used in OrderedSet structure.

Default None.

pipe(transaction=None)¶

Return a structure pipe given a transaction. The pipe is the Pipeline.pipe attribute of the structure pipeline.

Parameters:transaction – Optional stdnet.Transaction instance.
Return type:a local strcuture to hold data before it is safe into the backend database.
cursor(transaction=None)¶

Return the backend cursor given a transaction.

reload(transaction=None)¶

Fill data from the server

save(transaction=None)¶

Save the data in the back-end server. If a transaction is specified, the data is pipelined and executed when the transaction completes.

delete(transaction=None)¶

Delete the structure from the remote server. If a transaction is specified, the data is pipelined and executed when the transaction completes.

size(transaction=None)¶

Number of elements in structure. If no transaction is supplied, use the server default cursor.

has(value, transaction=None)¶

Check if value is in the structure. Pass a transaction if you would like ti pipeline the command.

add_expiry(cursor, timeout)¶

Internal method called if a timeout is set. This needs to implemented.

Set¶

class stdnet.Set(server, id, pipetype, instance=None, timeout=0, pickler=None, value_pickler=None, scorefun=None, **kwargs)¶

An unordered set stdnet.Structure. Equivalent to python set.

This structure is used for in two different parts of the library.

  • It is the structure upon which indexes are built, therefore each stdnet.orm.Field which has index set to True will have an associated Set structure in the data server backend.
  • It is also used as stdnet.orm.SetField.
add(value, transaction=None)¶

Add value to the set

update(values, transaction=None)¶

Add iterable values to the set

discard(value, transaction=None)¶

Remove an element from a set if it is a member

remove(values, transaction=None)¶

Remove elements from a set if they are members

HashTable¶

class stdnet.HashTable(server, id, pipetype, instance=None, timeout=0, pickler=None, value_pickler=None, scorefun=None, **kwargs)¶

A hash-table stdnet.Structure. The networked equivalent to a Python dict.

keyvaluedata(func, transaction, *args, **kwargs)¶

invoke remote function in the server and if we are not using a pipeline, unpickle the result.

add(key, value, transaction=None)¶

Add key - value pair to hashtable.

addnx(field, value, transaction=None)¶

Set the value of a hash field only if the field does not exist.

update(mapping, transaction=None)¶

Add mapping dictionary to hashtable. Equivalent to python dictionary update method.

Parameters:
  • mapping – a dictionary of field values,
  • transaction – a optional stadnet.Transaction instance.
get(key, default=None, transaction=None)¶

Retrieve a single element from the hashtable. If the element is not available return the default value (only when not using a transaction).

Parameters:
  • key – lookup field
  • default – default value when the field is not available.
  • transaction – an optional transaction instance.
Return type:

a value in the hashtable or a pipeline depending if a transaction has been used.
keys(desc=False)¶

Return a generator of all keys. No transactions involved.

items(keys=None)¶

Generator over key-values. If keys is not supplied, it is a generator over all key-value items. No transaction involved in this function.

range(start, end, desc=False)¶

Return a generator of ordered items between start and end.
