Backend Data Server¶

Behind the scenes is the database. Currently stdnet has support for Redis only but an open door is left for including other backends.

Backends¶

  • Redis Backend

API¶

Backend data server¶

class stdnet.BackendDataServer(name, address, pickler=None, charset='utf-8', connection_string='', prefix=None, **params)¶

Generic interface for a backend database:

Parameters:
  • name – name of database, such as redis, couchdb, etc..
  • params – dictionary of configuration parameters
  • pickler – calss for serializing and unserializing data.

It must implement the loads and dumps methods.

disconnect()¶

Disconnect the connection.

isempty()¶

Returns True if the database has no keys.

make_objects(meta, data, related_fields=None)¶

Generator of stdnet.odm.StdModel instances with data from database.

Parameters:
  • meta – instance of model stdnet.odm.Metaclass.
  • data – iterator over instances data.
structure(instance, client=None)¶

Create a backend stdnet.odm.Structure handler.

clean(meta)¶

Remove temporary keys for a model

basekey(meta, *args)¶

Calculate the key to access model data.

Parameters:
  • meta – a stdnet.odm.Metaclass.
  • args – optional list of strings which are attached to the basekey.
Return type:

a native string
setup_connection(address, **params)¶

Callback during initialization. Implementation should override this function for customizing their handling of connection parameters. It must return a instance of the backend handler.

execute_session(session, callback)¶

Execute a stdnet.odm.Session in the backend server.

model_keys(meta)¶

Return a list of database keys used by model model

instance_keys(obj)¶

Return a list of database keys used by instance obj

clear()¶

Remove all values from the database at once.

Backend Query¶

class stdnet.BackendQuery(queryelem, timeout=0, **kwargs)¶

Backend queryset class which implements the database queries specified by stdnet.odm.Query.

queryelem¶

The stdnet.odm.QueryElement to process.

executed¶

flag indicating if the query has been executed in the backend server
