Remote Structure¶

Data structures are fundamental constructs around which you build your application. They are used in almost all computer programs, therefore becoming fluent in what the standard data-structures can do for you is essential to get full value out of them. The standard template library in C++ implements a wide array of structures, python has several of them too. stdnet implements five remote structures, all derived from stdnet.Structure.

class stdnet.Structure(cursor, id, timeout=0, pickler=None, pipeline=None, scorefun=None, **kwargs)¶

Base class for remote data-structures.

cursor¶

instance of a stdnet.BackendDataServer.

id¶

unique id for the structure

timeout¶

Expiry timeout. If different from zero it represents the number of seconds after which the structure is deleted from the data server.

Default 0.

pickler¶

Class used for serialize and unserialize values. If None the cursor pickler will be used. If False no pikler will be used.

Default None.

pipeline¶

An instance of stdnet.PipeLine.

scorefun¶

A callable which takes a value as parameter and return a float number to be usedto score the value. Used in OrderedSet structure.

Default None.

size()¶

Number of elements in structure

delete()¶

Delete structure from remote server.

add_expiry()¶

Internal method called if a timeout is set. This needs to implemented.
