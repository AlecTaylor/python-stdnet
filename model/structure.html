Data-structure Fields¶

These fields are remote data-structures such as list, sets and hash tables. They can be bound to models so that many-to-many objects relationship can be established. All the data-structure fields derives from MultiField.

class stdnet.orm.MultiField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

Virtual class for data-structure fields:

relmodel¶

Optional stdnet.otm.StdModel class contained in the structure. It can also be specified as a string.

related_name¶

Same as stdnet.orm.ForeignKey Field.

pickler¶

a module/class/objects used to serialize values. Default None.

converter¶

a module/class/objects used to convert keys to suitable string to use as keys in stdnet.HashTable structures. It must implement two methods, tokey to convert key to a suitable key for the database backend and tovalue the inverse operation. Default: None.

ListField¶

class stdnet.orm.ListField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A field maintaining a list of values. When accessed from the model instance, it returns an instance of stdnet.List structure. For example:

class UserMessage(orm.StdModel):
    user = orm.SymbolField()
    messages = orm.ListField()

Lets register it with redis:

>>> orm.register(UserMessage,''redis://127.0.0.1:6379/?db=11')
'redis db 7 on 127.0.0.1:6379'

Can be used as:

>>> m = UserMessage(user = 'pippo').save()
>>> m.messages.push_back("adding my first message to the list")
>>> m.messages.push_back("ciao")
>>> m.save()
>>> type(u.messages)
<class 'stdnet.backends.structures.structredis.List'>
>>> u.messages.size()
2

SetField¶

class stdnet.orm.SetField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A field maintaining an unordered collection of values. It is initiated without any argument other than an optional model class. When accessed from the model instance, it returns an instance of stdnet.Set structure. For example:

class User(orm.StdModel):
    username  = orm.AtomField(unique = True)
    password  = orm.AtomField()
    following = orm.SetField(model = 'self')

It can be used in the following way:

>>> user = User(username = 'lsbardel', password = 'mypassword').save()
>>> user2 = User(username = 'pippo', password = 'pippopassword').save()
>>> user.following.add(user2)
>>> user.save()
>>> user2 in user.following
True

HashField¶

class stdnet.orm.HashField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A Hash table field, the networked equivalent of a python dictionary. Keys are string while values are string/numeric. it returns an instance of stdnet.HashTable structure.

ManyToManyField¶

class stdnet.orm.ManyToManyField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A many-to-many relationship. It accepts related_name as extra argument.

related_name¶

Optional name to use for the relation from the related object back to self.

For example:

class User(orm.StdModel):
    name      = orm.SymbolField(unique = True)
    following = orm.ManyToManyField(model = 'self',
                                    related_name = 'followers')

To use it:

>>> u = User(name = 'luca').save()
>>> u.following.add(User(name = 'john').save())
>>> u.following.add(User(name = 'mark').save())

This field is implemented as a double Set field.
