StdNet Models¶

The StdNet Object Relational Mapper presents a method of associating user-defined Python classes, referred as models, with stdnet.HashTable structures in a stdnet.BackendDataServer. These python classes are referred as models and are subclasses of stdnet.orm.StdModel.

Each instance of model represents a key-value in the corresponding stdnet.HashTable.

class stdnet.orm.StdModel(**kwargs)¶

A model is the single, definitive source of data about your data. It contains the essential fields and behaviors of the data you’re storing. Each model class maps to a single stdnet.HashTable structure via the StdModel._meta attribute.

_meta¶

Instance of stdnet.orm.base.Metaclass

customAttribute(name)¶

Override this function to provide custom attributes

save(commit=True)¶

Save the instance in the remote stdnet.HashTable The model must be registered with a stdnet.backends.BackendDataServer otherwise a stdnet.exceptions.ModelNotRegistered exception will raise.

delete(dlist=None)¶

Delete an instance from database. If the instance is not available (it does not have an id) and StdNetException exception will raise.

related_objects()¶

Collect or related objects

model_to_dict(fields=None, exclude=None)¶

Convert self to a dictionary.

Creating Models¶

Defining a stdnet models is simple, you derive a Python class from StdModel:

from stdnet import orm

class Author(orm.StModel):
    name = orm.AtomField()

class Book(orm.StdModel):
    title  = orm.AtomField()
    author = orm.ForeignKey(Author, related_name = 'books')

Register A Model¶

Once a model is defined, in order to use it in an application it needs to be registered with a back-end database.

stdnet.orm.register(model, backend=None, keyprefix=None, timeout=0)¶

Register a stdnet.orm.StdModel model with a stdnet.backends.BackendDataServer data server.

Parameters:
  • model – a stdnet.orm.StdModel class. Must be provided.
  • backend – a backend connection string. Default settings.DEFAULT_BACKEND.
  • keyprefix – a string used to prefix all database keys. Default settings.DEFAULT_KEYPREFIX.
  • keyprefix – timeout in seconds for keys persistance. Default 0 (no timeout).

Usage

For Redis the syntax is the following:

import orm

orm.register(Author, 'redis://my.host.name:6379/?db=1')
orm.register(Book, 'redis://my.host.name:6379/?db=2')

my.host.name can be localhost or an ip address while db indicate the database number (very useful for separating data on the same redis instance).

Data Server Metaclass¶

class stdnet.orm.base.Metaclass(model, fields, abstract=False, keyprefix=None, app_label='', verbose_name=None, **kwargs)¶

Utility class used for storing all information which maps a stdnet.orm.StdModel model into a stdnet.HashTable structure in a stdnet.BackendDataServer. An instance is initiated when stdnet.orm.StdModel class is created:

model¶

a subclass of stdnet.orm.StdModel.

fields¶

dictionary of stdnet.orm.Field instances.

abstract¶

if True, it represents an abstract model and no database elements are created.

keyprefix¶

prefix for the database table. By default it is given by settings.DEFAULT_KEYPREFIX, where settings is obtained by:

from dynts.conf import settings
pk¶

primary key :stdnet.orm.Field

basekey(*args)¶

Calculate the key to access model hash-table, and model filters in the database. For example:

>>> a = Author(name = 'Dante Alighieri').save()
>>> a.meta.basekey()
'stdnet:author'
uniqueid¶

Unique id for an instance. This is unique across multiple model types.

table()¶

Return an instance of stdnet.HashTable holding the model table

make(id, data)¶

Create a model instance from server data
