Fields API¶

The most important part of a model, and the only required part of a model, is the list of database fields it defines. Fields are specified by class attributes. They are the equivalent to django Fields and therefore the equivalent of columns in a traditional relational databases.

Note

There is an important difference between stdnet fields and fields in a traditional relational database. Stdnet fields can be added, removed or changed without requiring any time consuming database migration!

This is an added bonus which can be of high importance when prototyping but it can lead to loss of data if a user abuses it.

Base Classes¶

The Field class is the base class for all fields and contains all the attribute and method definitions. Derived classes only implement a handful of methods.

Field¶

class stdnet.orm.Field(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

This is the base class of all StdNet Fields. Each field is specified as a stdnet.orm.StdModel class attribute.

index¶

If True, the field will create indexes for queries. An index is implemented as a stdnet.Set in the stdnet.BackendDataServer. if the stdnet.orm.base.Metaclass.ordering attribute is set the indexes are implemented using stdnet.OrderedSet.

If you don’t need to query the field you should set this value to False, it will save you memory.

Note

if index is set to False executing queries against the field will throw a stdnet.QuerySetError exception. No database queries are allowed for non indexed fields as a design decision (excplicit better than implicit).

Default True.

unique¶

If True, the field must be unique throughout the model. In this case Field.index is also True. Enforced at stdnet.BackendDataServer level.

Default False.

primary_key¶

If True, this field is the primary key for the model. In this case Field.unique is also True.

Default False.

required¶

If False, the field is allowed to be null.

Default True.

default¶

Default value for this field. It can be a callable attribute.

Default None.

name¶

Field name, created by the orm at runtime.

model¶

The stdnet.orm.StdModel holding the field. Created by the orm at runtime.

charset¶

The charset used for encoding decoding text.

hidden¶

If True the field will be hidden from search algorithms.

Default False.

to_python(value)¶

Converts the input value into the expected Python data type, raising stdnet.FieldValueError if the data can’t be converted. Returns the converted value. Subclasses should override this.

register_with_model(name, model)¶

Called during the creation of a the stdnet.orm.StdModel class when stdnet.orm.base.Metaclass is initialised. It fills Field.name and Field.model. This is an internal function users should never call.

serialize(value, transaction=None)¶

Called by the stdnet.orm.StdModel.save() method when saving an object to the remote data server. It returns a representation of value to store in the database. If an error occurs it raises stdnet.exceptions.FieldValueError

id(obj)¶

Field id for object obj, if applicable. Default is None.

get_default()¶

Returns the default value for this field.

index_value()¶

A value which is used by indexes to generate keys.

scorefun(value)¶

Function which evaluate a score from the field value. Used by the ordering alorithm

AtomField¶

class stdnet.orm.AtomField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

The base class for fields containing atoms. An atom is an irreducible value with a specific data type. it can be of four different types:

  • boolean
  • integer
  • date
  • datetime
  • floating point
  • symbol

MultiField¶

class stdnet.orm.MultiField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

Virtual class for data-structure fields:

relmodel¶

Optional stdnet.otm.StdModel class contained in the structure. It can also be specified as a string.

related_name¶

Same as stdnet.orm.ForeignKey Field.

pickler¶

a module/class/objects used to serialize values.

Default None.

converter¶

a module/class/objects used to convert keys to suitable string to use as keys in stdnet.HashTable structures. It must implement two methods, tokey to convert key to a suitable key for the database backend and tovalue the inverse operation.

Default: None.

Atom Fields¶

Atom Fields derived from stdnet.orm.AtomField and, as the name says, they represent the simplest data in a model. Their representation in python, is one of bytes, strings, numbers or dates.

IntegerField¶

class stdnet.orm.IntegerField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An integer AtomField.

BooleanField¶

class stdnet.orm.BooleanField(required=False, **kwargs)¶

A boolean AtomField

AutoField¶

class stdnet.orm.AutoField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An IntegerField that automatically increments. You usually won’t need to use this directly; a primary_key field of this type, named id, will automatically be added to your model if you don’t specify otherwise.

FloatField¶

class stdnet.orm.FloatField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An floating point AtomField. By default its Field.index is set to False.

SymbolField¶

class stdnet.orm.SymbolField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An AtomField which contains a symbol. A symbol holds a unicode string as a single unit. A symbol is irreducible, and are often used to hold names, codes or other entities. They are indexes by default.

CharField¶

class stdnet.orm.CharField(*args, **kwargs)¶

A text SymbolField which is never an index. It contains unicode and by default and Field.required is set to False.

It accept an additional attribute

as_cache¶

If True the field contains cached data.

Default False.

ByteField¶

class stdnet.orm.ByteField(*args, **kwargs)¶

A field which contains binary data. In python this is converted to bytes.

DateField¶

class stdnet.orm.DateField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An AtomField represented in Python by a datetime.date instance.

DateTimeField¶

class stdnet.orm.DateTimeField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

An date AtomField represented in Python by a datetime.datetime instance.

Object Type Fields¶

ForeignKey¶

class stdnet.orm.ForeignKey(model, related_name=None, **kwargs)¶

A field defining a one-to-many objects relationship. Requires a positional argument: the class to which the model is related. To create a recursive relationship, an object that has a many-to-one relationship with itself use:

orm.ForeignKey('self')

It accepts related_name as extra argument. It is the name to use for the relation from the related object back to self. For example:

class Folder(orm.StdModel):
    name = orm.SymobolField()

class File(orm.StdModel):
    folder = orm.ForeignKey(Folder, related_name = 'files')

JSONField¶

class stdnet.orm.JSONField(*args, **kwargs)¶

A JSON field which implements automatic conversion to and from an object and a JSON string. It is the responsability of the user making sure the object is JSON serializable.

There are few extra parameters which can be used to customize the behaviour and how the field is stored in the back-end server.

Parameters:
  • encoder_class –

    The JSON class used for encoding.

    Default: stdnet.utils.jsontools.JSONDateDecimalEncoder.

  • decoder_hook –

    A JSON decoder function.

    Default: stdnet.utils.jsontools.date_decimal_hook.

  • nested –

    If True a nested JSON dictionary is build by separating fields with the __ double underscore separator. This flag is used when saving data to the database.

    Default None.

  • as_string –

    a flag indicating if data should be serialized into a JSON string. If the value is set to False the JSON data is stored as a field of the instance prefixed with the field name and double underscore __. If True it is stored as a standard JSON string on the field.

    Default True.

For example, lets consider the following:

class MyModel(orm.StdModel):
    name = orm.SymbolField()
    data = orm.JSONField(as_string = False)

And:

>>> m = MyModel(name='bla',pv={'': 0.5, 'mean': 1, 'std': 3.5})
>>> m.cleaned_data
{'name': 'bla', 'pv': 0.5, 'pv__mean': '1', 'pv__std': '3.5'}
>>>

The reason for setting as_string to False is to enable sorting of instances with respect to its fields:

>>> MyModel.objects.all().sort_by('data__pv__std')
>>> MyModel.objects.all().sort_by('-data__pv')

which can be rather useful feature.

PickleObjectField¶

class stdnet.orm.PickleObjectField(*args, **kwargs)¶

A field which implements automatic conversion to and form a pickable python object. This field is python specific and therefore not of much use if accessed from external programs. Consider the ForeignKey or JSONField fields as more general alternatives.

ModelField¶

class stdnet.orm.ModelField(unique=False, ordered=None, primary_key=False, required=True, index=None, hidden=None, **extras)¶

A filed which can be used to store the model classes (not only stdnet.orm.StdModel models). If a class has a attribute _meta with a unique hash attribute hash and it is registered in the model hash table, it can be used.

Data-structure Fields¶

These fields are remote data-structures such as list, sets and hash tables. They can be bound to models so that many-to-many objects relationship can be established. All the data-structure fields derives from stdnet.orm.MultiField.

ListField¶

class stdnet.orm.ListField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A field maintaining a list of values. When accessed from the model instance, it returns an instance of stdnet.List structure. For example:

class UserMessage(orm.StdModel):
    user = orm.SymbolField()
    messages = orm.ListField()

Lets register it with redis:

>>> orm.register(UserMessage,''redis://127.0.0.1:6379/?db=11')
'redis db 7 on 127.0.0.1:6379'

Can be used as:

>>> m = UserMessage(user = 'pippo').save()
>>> m.messages.push_back("adding my first message to the list")
>>> m.messages.push_back("ciao")
>>> m.save()
>>> type(u.messages)
<class 'stdnet.backends.structures.structredis.List'>
>>> u.messages.size()
2

SetField¶

class stdnet.orm.SetField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A field maintaining an unordered collection of values. It is initiated without any argument other than an optional model class. When accessed from the model instance, it returns an instance of stdnet.Set structure. For example:

class User(orm.StdModel):
    username  = orm.AtomField(unique = True)
    password  = orm.AtomField()
    following = orm.SetField(model = 'self')

It can be used in the following way:

>>> user = User(username = 'lsbardel', password = 'mypassword').save()
>>> user2 = User(username = 'pippo', password = 'pippopassword').save()
>>> user.following.add(user2)
>>> user.save()
>>> user2 in user.following
True

HashField¶

class stdnet.orm.HashField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A Hash table field, the networked equivalent of a python dictionary. Keys are string while values are string/numeric. it returns an instance of stdnet.HashTable structure.

ManyToManyField¶

class stdnet.orm.ManyToManyField(model=None, pickler=None, converter=None, required=False, related_name=None, scorefun=None, **kwargs)¶

A many-to-many relationship. It accepts related_name as extra argument.

related_name¶

Optional name to use for the relation from the related object back to self.

For example:

class User(orm.StdModel):
    name      = orm.SymbolField(unique = True)
    following = orm.ManyToManyField(model = 'self',
                                    related_name = 'followers')

To use it:

>>> u = User(name = 'luca').save()
>>> u.following.add(User(name = 'john').save())
>>> u.following.add(User(name = 'mark').save())

and to remove:

>>> u.following.remove(User.objects.get(name = 'john))

This field is implemented as a double Set field.
